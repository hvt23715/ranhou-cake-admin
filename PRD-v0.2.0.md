## 1. 项目概述

### 1.1 项目背景

- **客户**：燃厚蛋糕
  - 吉林省长春市头部烘焙品牌
  - 约20家直营门店
  - 隶属于吉林省九亿食品有限公司
- **目标**：结合店大师能力，为燃厚蛋糕设计 B端管理系统
- **定位**：店大师第一次 ToB 尝试，打造烘焙行业数字化管理标杆案例
- **意义**：探索店大师 SaaS 能力在垂直行业的应用场景

### 1.2 产品定位

面向烘焙连锁品牌的**智能经营管理平台**，以 AI 问答为核心交互方式，融合地理数据洞察与行业知识库，为老板/管理者提供决策支持。

### 1.3 设计理念

```
┌─────────────────────────────────────────────────────────────┐
│                      以 AI 问答为中心                          │
│                                                              │
│  数据查询  ←→  行业知识  ←→  地理洞察  ←→  预警提醒        │
│                                                              │
│                      + 固定数据看板                           │
│                      + 实时运营监控                           │
└─────────────────────────────────────────────────────────────┘
```

### 1.4 数据来源架构

| 数据类型 | 来源 | 说明 |
|----------|------|------|
| **销售数据** | 静态 Demo 数据 | PRD 中预设的模拟数据 |
| **门店数据** | 静态 Demo 数据 | 20 家门店的名称、地址、坐标 |
| **品类数据** | 静态 Demo 数据 | 面包、蛋糕、饮品等分类销量 |
| **天气数据** | 静态 Demo 数据 | 未来 7 天天气预报 |
| **周边 POI** | 高德地图 API | 实时查询竞品、交通设施等 |
| **客流数据** | 静态 Demo 数据 | Demo 模拟，未来可接入门店摄像头统计 |
| **违规检测** | 静态 Demo 数据 | Demo 模拟，未来可接入 AI 摄像头 |

## 4. 功能模块

### 4.1 AI 问答中心（核心功能 ⭐）

**设计理念**：像和业务助手聊天一样管理门店，通过自然语言获取数据和建议。

#### 4.1.1 问答交互界面

- **位置**：首页入口 + 独立全屏页面
- **交互方式**：
  - 文字输入（支持回车发送）
  - 点击建议问题（快捷入口）
- **回复展示**：
  - 纯文本回复
  - 数据表格（Markdown 表格）
  - 图表（折线图、柱状图等）
  - 建议卡片（带图标的操作建议）

#### 4.1.2 建议问题分类

| 类别 | 图标 | 示例问题 | 数据来源 |
|------|------|----------|----------|
| **数据查询** | 📊 | "今天全部门店的销售额是多少？" | 静态数据 |
| | 📊 | "上周销量最高和最低的门店分别是哪家？" | 静态数据 |
| | 📊 | "本月各品类的销量占比是多少？" | 静态数据 |
| | 📊 | "对比一下红旗街万达店和欧亚卖场店的本周销量" | 静态数据 |
| **运营建议** | 💡 | "哪些单品最近3天销量持续下降？" | 静态数据 |
| | 💡 | "帮我分析一下上周的销售数据，有什么问题？" | 静态数据 |
| | 💡 | "根据近期数据，给出下周的备货建议" | 静态数据 |
| | 💡 | "下午时段客流低，有什么提升建议？" | 行业知识 |
| **行业知识** | 📚 | "面包店如何提升下午时段的客流？" | 行业知识库 |
| | 📚 | "生日蛋糕的库存周转率一般多少正常？" | 行业知识库 |
| | 📚 | "烘焙行业最近有什么新趋势？" | 行业知识库 |
| | 📚 | "新店开业前三个月应该注意什么？" | 行业知识库 |
| **违规检测** | 🔍 | "今天有门店出现食品安全违规情况吗？" | 静态数据 |
| | 🔍 | "哪些门店的卫生评分下降了？" | 静态数据 |
| | 🔍 | "员工操作规范方面有什么问题？" | 静态数据 |
| **地理洞察** | 🗺️ | "分析一下各门店周边的竞争对手情况" | **高德API + 静态数据** |
| | 🗺️ | "哪些门店选址附近的客流密度最高？" | 静态数据 |
| | 🗺️ | "基于门店分布，给出最优配送路线建议" | 高德API |
| | 🗺️ | "红旗街万达店周边500米有多少竞品？" | **高德API 实时查询** |
| | 🗺️ | "这家店附近有哪些地铁站和公交站？" | **高德API 实时查询** |

#### 4.1.3 AI 能力融合

| 能力 | 数据来源 | 实现方式 |
|------|----------|----------|
| **销售数据分析** | 静态 Demo 数据 | 直接从本地 JSON 数据分析 |
| **行业知识问答** | 内置知识库 | Prompt 中内置烘焙行业知识 |
| **周边 POI 查询** | 高德地图 API | 实时调用 `https://restapi.amap.com/v3/place/around` |
| **竞品分析** | 高德地图 API | 搜索周边烘焙店类型 |
| **路径规划** | 高德地图 API | 调用 `/v3/direction` 接口 |
| **天气预报** | 静态 Demo 数据 | Demo 阶段用预设数据 |

### 4.3 最新动态预警

**位置**：侧边栏 + 首页卡片

#### 4.3.1 预警类型

| 类型 | 图标 | 触发条件 | 示例内容 | 数据来源 |
|------|------|----------|----------|----------|
| 🔥 爆单预警 | 🔥 | 单品1小时订单量 > 50 | "【爆单】芒果慕斯最近1小时订单量激增150%" | 静态数据 |
| ⚠️ 滞销预警 | ⚠️ | 单品连续3天销量 < 5 | "【滞销】提拉米苏已连续3天日均销量不足5件" | 静态数据 |
| 📈 门店增长 | 📈 | 单店日销量环比 > 50% | "【增长】红旗街万达店今日销量环比+68%" | 静态数据 |
| 📉 门店下滑 | 📉 | 单店日销量环比 < -30% | "【下滑】欧亚卖场店今日销量环比-42%" | 静态数据 |
| 🆕 新品表现 | 🆕 | 上新7天内单品 | "【新品】草莓芝士蛋糕上市5天，日均销量32件" | 静态数据 |
| 🎯 目标达成 | 🎯 | 门店/品类销售目标进度 | "【目标】面包品类本周完成率78%，还需1.2万" | 静态数据 |

#### 4.3.2 预警展示规则

- **排序**：时间倒序，最新在上
- **筛选**：按类型筛选（全部/爆单/滞销/增长/下滑/新品/目标）
- **状态**：未读/已处理
- **操作**：点击标记为已处理

### 4.4 地图热力展示

**技术**：高德地图 JavaScript API + 数据可视化插件

#### 4.4.1 门店分布地图

- **底图**：高德地图标准底图
- **标记**：20 个门店标记点（AMap.Marker）
- **弹窗**：点击显示门店名称、地址、今日销量、实时客流
- **交互**：缩放、拖拽、点击查看详情

#### 4.4.2 销量热力

- **热力图层**：AMap.Heatmap
- **数据**：20 家门店的今日销量
- **颜色梯度**：红（高销量）→ 橙 → 黄 → 绿 → 蓝（低销量）
- **筛选**：今日 / 本周 / 本月

#### 4.4.3 客流热力

- **热力图层**：AMap.Heatmap
- **数据**：20 家门店的今日客流
- **颜色梯度**：同销量热力
- **对比**：可切换显示销量热力或客流热力，分析转化率

#### 4.4.4 地理洞察（结合高德 API）

| 功能 | 说明 | 数据来源 |
|------|------|----------|
| 门店分布概览 | 查看所有门店的地理分布 | 静态坐标数据 |
| 周边竞品分析 | 门店周边 500m 内的烘焙店数量 | **高德 API 实时查询** |
| 交通设施 | 周边地铁站、公交站数量 | **高德 API 实时查询** |
| 客流预测 | 基于位置的人口密度分析 | 静态数据（Demo） |

|------|------|
| POI | Point of Interest，兴趣点（地图上的地标、建筑、商户） |

| 同/环比 | 与上一期/同去年相比的增长/下降 |
