# 燃厚蛋糕B端管理系统 - 软件说明书

> **软件名称**：燃厚蛋糕B端管理系统
> **版本**：V1.0.0
> **开发完成日期**：2026年3月
> **著作权人**：吉林省九亿食品有限公司

---

## 一、软件概述

### 1.1 项目背景

- **客户**：燃厚蛋糕（吉林省长春市头部烘焙品牌，约20家直营门店）
- **目标**：结合店大师能力，为燃厚蛋糕设计B端管理系统
- **定位**：面向烘焙连锁品牌的智能经营管理平台，以AI问答为核心交互方式，为管理者提供决策支持

### 1.2 设计理念

```
┌─────────────────────────────────────────────────────────────┐
│                      以 AI 问答为中心                          │
│                                                              │
│  数据查询  ←→  行业知识  ←→  地理洞察  ←→  预警提醒        │
│                                                              │
│                      + 固定数据看板                           │
│                      + 实时运营监控                           │
└─────────────────────────────────────────────────────────────┘
```

## 二、功能架构

### 2.1 系统功能模块

| 序号 | 模块名称 | 功能描述 | 优先级 |
|------|----------|----------|--------|
| 1 | AI问答中心 | 核心功能，通过自然语言对话实现数据查询、运营建议、行业知识问答、违规检测、地理洞察 | P0 |
| 2 | 数据看板 | 首页仪表盘，展示核心指标、销量趋势、品类排行、门店排行、时段分布 | P0 |
| 3 | 预警系统 | 实时推送爆单预警、滞销预警、门店增长/下滑、新品表现、目标达成等动态 | P0 |
| 4 | 地图热力 | 基于高德地图展示门店分布、销量热力、客流热力、周边POI分析 | P0 |
| 5 | 摄像头监控 | 实时查看门店摄像头画面，AI识别违规情况（Demo阶段使用模拟数据） | P1 |

### 2.2 数据来源架构

| 数据类型 | 来源 | 说明 |
|----------|------|------|
| 销售数据 | 静态Demo数据 | PRD中预设的模拟数据 |

| 品类数据 | 静态Demo数据 | 面包、蛋糕、饮品等分类销量 |
| 天气数据 | 静态Demo数据 | 未来7天天气预报 |

## 三、功能详细说明

### 3.1 AI问答中心

#### 3.1.1 问答交互界面

- **交互方式**：文字输入（支持回车发送）、点击建议问题（快捷入口）
- **回复展示**：纯文本回复、数据表格（Markdown表格）、图表（折线图、柱状图等）、建议卡片

#### 3.1.2 建议问题分类

| 类别 | 示例问题 | 数据来源 |
|------|----------|----------|
| 数据查询 | "今天全部门店的销售额是多少？" | 静态数据 |
| 数据查询 | "上周销量最高和最低的门店分别是哪家？" | 静态数据 |
| 数据查询 | "本月各品类的销量占比是多少？" | 静态数据 |
| 数据查询 | "对比一下红旗街万达店和欧亚卖场店的本周销量" | 静态数据 |
| 运营建议 | "哪些单品最近3天销量持续下降？" | 静态数据 |
| 运营建议 | "帮我分析一下上周的销售数据，有什么问题？" | 静态数据 |
| 运营建议 | "根据近期数据，给出下周的备货建议" | 静态数据 |
| 运营建议 | "下午时段客流低，有什么提升建议？" | 行业知识 |
| 行业知识 | "面包店如何提升下午时段的客流？" | 行业知识库 |
| 行业知识 | "生日蛋糕的库存周转率一般多少正常？" | 行业知识库 |
| 行业知识 | "烘焙行业最近有什么新趋势？" | 行业知识库 |
| 违规检测 | "今天有门店出现食品安全违规情况吗？" | 静态数据 |
| 违规检测 | "哪些门店的卫生评分下降了？" | 静态数据 |
| 违规检测 | "员工操作规范方面有什么问题？" | 静态数据 |
| 地理洞察 | "分析一下各门店周边的竞争对手情况" | 高德API+静态数据 |
| 地理洞察 | "哪些门店选址附近的客流密度最高？" | 静态数据 |
| 地理洞察 | "基于门店分布，给出最优配送路线建议" | 高德API |
| 地理洞察 | "红旗街万达店周边500米有多少竞品？" | 高德API实时查询 |

#### 3.1.3 AI能力融合

| 能力 | 数据来源 | 实现方式 |
|------|----------|----------|
| 销售数据分析 | 静态Demo数据 | 直接从本地JSON数据分析 |

### 3.3 预警系统

#### 3.3.1 预警类型

| 类型 | 触发条件 | 示例内容 |
|------|----------|----------|
| 爆单预警 | 单品1小时订单量 > 50 | "【爆单】芒果慕斯最近1小时订单量激增150%" |
| 滞销预警 | 单品连续3天销量 < 5 | "【滞销】提拉米苏已连续3天日均销量不足5件" |
| 门店增长 | 单店日销量环比 > 50% | "【增长】红旗街万达店今日销量环比+68%" |
| 门店下滑 | 单店日销量环比 < -30% | "【下滑】欧亚卖场店今日销量环比-42%" |
| 新品表现 | 上新7天内单品 | "【新品】草莓芝士蛋糕上市5天，日均销量32件" |
| 目标达成 | 门店/品类销售目标进度 | "【目标】面包品类本周完成率78%，还需1.2万" |

#### 3.3.2 预警展示规则

- **排序**：时间倒序，最新在上
- **筛选**：按类型筛选（全部/爆单/滞销/增长/下滑/新品/目标）
- **状态**：未读/已处理
- **操作**：点击标记为已处理

### 3.4 地图热力

#### 3.4.1 门店分布地图

- **底图**：高德地图标准底图
- **标记**：20个门店标记点
- **弹窗**：点击显示门店名称、地址、今日销量、实时客流
- **交互**：缩放、拖拽、点击查看详情

#### 3.4.2 销量/客流热力

- **热力图层**：高德地图热力图插件
- **数据**：20家门店的今日销量/客流
- **颜色梯度**：红（高销量）→ 橙 → 黄 → 绿 → 蓝（低销量）
- **筛选**：今日 / 本周 / 本月

#### 3.4.3 地理洞察

| 功能 | 说明 | 数据来源 |
|------|------|----------|
| 门店分布概览 | 查看所有门店的地理分布 | 静态坐标数据 |
| 周边竞品分析 | 门店周边500m内的烘焙店数量 | 高德API实时查询 |
| 交通设施 | 周边地铁站、公交站数量 | 高德API实时查询 |
