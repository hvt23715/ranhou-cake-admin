## 产品概述

燃厚蛋糕管理系统首页功能增强，将原有的功能入口卡片升级为可直接交互的功能预览模块。

## 核心功能需求

1. **首页AI智能问答组件**

- 在首页 Dashboard 集成简化版 AI 问答功能
- 支持用户直接输入问题并获取回答
- 复用 AIAssistant 页面的 `generateResponse` 逻辑
- 显示最近对话历史（3-5条消息）

2. **首页地图预览组件**

- 展示门店分布缩略图
- 使用 ECharts 散点图模拟地图效果（基于门店经纬度数据）
- 点击可跳转至完整地图页面

3. **首页摄像头预览组件**

- 展示3个摄像头的实时预览缩略图
- 显示门店名称和摄像头位置
- 点击跳转至摄像头监控页面

4. **门店地图热力图功能**

- 在 StoreMap 页面添加热力图层
- 支持销量热力图和客流热力图两种模式切换
- 使用高德地图热力图插件实现

## 技术栈

- React 18 + TypeScript + Vite
- Tailwind CSS 样式
- ECharts 图表库 (echarts-for-react)
- 高德地图 JavaScript API
- Lucide React 图标

## 实现方案

### 架构设计

采用组件化设计，将 Dashboard 中的三个功能预览拆分为独立组件：

```
Dashboard (页面)
├── DashboardAIChat (首页AI问答)
├── DashboardMapPreview (首页地图预览)
├── DashboardCameraPreview (首页摄像头预览)
└── 原有统计图表组件
```

### 关键实现细节

1. **AI 问答组件**

- 提取 `generateResponse` 函数到独立工具模块，供 Dashboard 和 AIAssistant 共用
- 使用 `useState` 管理消息列表（限制最多5条）
- 消息气泡样式与 AIAssistant 页面保持一致

2. **地图预览组件**

- 使用 ECharts 散点图 (`type: 'scatter'`) 基于经纬度渲染门店分布
- 点大小映射销量/客流量
- 添加点击事件跳转至 StoreMap 页面

3. **摄像头预览组件**

- 复用 CameraMonitor 的视觉样式（深色背景 + LIVE 标识）
- 选择3个代表性门店展示
- 使用网格布局展示3个缩略图

4. **热力图功能**

- 使用高德地图 `AMap.Heatmap` 插件
- 创建 `heatType` 状态控制切换（sales | customers）
- 热力数据格式：`{ lng, lat, count }`
- 添加切换按钮组件

### 性能优化

- Dashboard 组件使用懒加载（非首屏关键）
- 地图预览使用静态散点图而非真实地图 API，减少首页加载时间
- 热力图数据在 StoreMap 组件内计算，避免重复遍历

## 设计架构

首页 Dashboard 采用三栏布局展示三个功能预览模块：

### 布局结构

- 顶部：统计卡片（保持不变）
- 中部：功能预览区（3列等宽布局）
- 左栏：AI智能助手（占1列，高度约320px）
- 中栏：门店分布预览（占1列，高度约320px）
- 右栏：摄像头监控（占1列，高度约320px）
- 底部：原有图表区域

### 功能预览区设计

**AI智能助手卡片**

- 顶部：标题 "AI 智能助手" + 跳转箭头
- 中部：消息区域（最大显示3条消息，带滚动）
- 用户消息：右对齐，蓝色背景
- AI消息：左对齐，灰色背景，带机器人图标
- 底部：输入框 + 发送按钮（单行紧凑布局）

**门店分布预览卡片**

- 顶部：标题 "门店分布" + 跳转箭头
- 中部：ECharts 散点图
- 使用门店经纬度数据
- 点颜色映射销量（渐变色）
- 悬停显示门店名称和销售额
- 底部：门店数量统计

**摄像头监控预览卡片**

- 顶部：标题 "实时监控" + 跳转箭头
- 中部：3个摄像头缩略图网格
- 深色背景（bg-gray-900）
- LIVE 红色标识（带脉冲动画）
- 底部显示门店和位置信息
- 底部：未处理预警数量

### StoreMap 热力图设计

- 地图右上角添加切换控件（Segmented Control）
- 两个选项："销量热力" / "客流热力"
- 选中态：主色背景 + 白色文字
- 热力图颜色渐变：蓝 → 绿 → 黄 → 红
